# [multi-frp v0.2.0](https://github.com/Locietta/multi-frp/releases/tag/v0.2.0) - 2026-01-14

* Improved CLI, now use `multi-frp -c <config.json>` to specify config file
* refactor to improve code quality (eliminate global vars etc.)
* remove misleading relative path resolution
* remove signal handling on windows (we really don't need it)
* improve signal handling on linux, avoid using signal unsafe handler
* disable upx

**Full Changelog**: https://github.com/Locietta/multi-frp/compare/v0.1.0...v0.2.0

# [multi-frp v0.1.0](https://github.com/Locietta/multi-frp/releases/tag/v0.1.0) - 2026-01-13

* Initial release
* Launches frpc binary with each specified config
* Kill the frpc process it launches when itself dies, this includes:
  * keyboard interuptions from terminal
  * (on windows) task manager's "end task" event
